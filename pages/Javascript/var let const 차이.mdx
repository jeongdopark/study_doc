> [!note] const, let, var 차이점

ES6 이전에는 변수를 선언할 수 있는 방법은 var 키워드를 사용하는 것뿐이었다.
ES6부터 let과 const 키워드가 등장하였다.
변수 선언하는데 사용하는 var, let, const의 차이점에 대해서 알아보자.

## 1. 중복 선언



- **var** 키워드는 중복 선언이 가능하다.

```js
var test = 1;
console.log(test); // 1

var test = 2;
console.log(test); // 2
```

- **let**과 **const** 키워드는 중복 선언이 불가능하다.

## 2. Scope

- var 키워드는 함수 레벨 스코프만을 지역 스코프로 인정한다.
- let과 const 키워드는 블록 레벨 스코프를 따른다.

## 3. Hoisting

### var

변수 호이스팅에 의해 변수 선언문 이전에 참조할 수 있다.
단, 할당문 이전에 변수를 참조하면 언제나 'undefined'를 반환한다.
var 키워드로 선언한 변수는 런타임 이전에 자바스크립트 엔진에 의해 "선언 단계"와 "초기화 단계"가 한번에 진행된다.

### let과 const

let과 const 키워드로 선언한 변수는 "선언 단계"와 "초기화 단계"가 분리되어 진행된다.
즉, 런타임 이전에 자바스크립트 엔진에 의해 선언 단계가 먼저 진행되고, 초기화 단계는 변수 선언문에 도달했을 때 실행된다. 이전에 변수에 접근하면 참조 에러 ( Reference Error )가 발생한다.
스코프의 시작 지점부터 초기화 시작 지점까지 변수를 참조할 수 없는 구간을 "**일시적 사각지대 TDZ**"라고 부른다.

변수 호이스팅이 발생하지 않는 것처럼 보이지만, 아니다.
만약 호이스팅이 발생하지 않는다면 아래 코드에서 참조 에러가 발생하지 않아야한다.
참조 에러가 발생하는 것이 const와 let도 호이스팅이 발생한다는 의미를 뜻한다.

```js
const test = 1;

{
  console.log(test); // Reference Error
  const test = 2;
}
```

#### 재할당 금지

const 키워드로 선언된 변수에 원시값이 할당되었을 경우 값을 변경할 수 없다.
하지만, 객체를 할당할 경우 값을 변경할 수 있다.
객체는 재할당 없이도 값 변경이 가능하기 때문이다.
즉, const는 **재할당을 금지**하는 것이지 **불변**을 의미하지 않는다.

# 실행 컨텍스트란 ? ( Execution Context )

> **실행할 코드의** 환경 정보들을 모아놓은 객체이다.

동일한 스코프에 있는 코드들을 실행할 때 필요한 환경 정보들을 모아 컨텍스트를 구성하고, 이를 콜 스택에 쌓아올렸다가, 가장 위에 있는 컨텍스트가 실행되는 식으로 전체 코드의 환경과 순서를 보장한다.

하나의 컨텍스트를 구성할 수 있는 방법으로 전역 공간, eval함수, 함수, 모듈이 있다.
하지만 흔히 실행 컨텍스트를 구성하는 방법은 함수를 실행하는 것 뿐이다.

전역 공간에 의해서 실행 컨텍스트가 구성되는 것은 코드 내부의 별도 명령 없이 자바스크립트 파일이 열리는 순간 자동으로 전역 컨텍스트가 활성화되는 것으로 이해하면 된다.

환경 정보를 수집해서 실행 컨텍스트 객체에 저장하는데, 다음과 같은 정보가 저장된다.

1. Variable Environment
   - 현재 컨텍스트 내의 식별자들에 대한 정보와 외부 환경 정보 저장.
   - 선언 시점의 Lexical Environment의 스냅샷으로 변경되지 않는다.
2. Lexical Environment
   - 최초 Variable Environment와 같지만 변경 사항 실시간으로 반영된다.
3. This Binding
   - this 식별자가 바라봐야 할 대상 객체.
